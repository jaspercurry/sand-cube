# CAD Troubleshooting Notes

This project uses Build123d/OpenCascade to generate exact CAD, then imports the
STEP files into Onshape for review. A model can be a valid OpenCascade solid and
still import oddly in Onshape, especially around thickened, revolved spline
surfaces. This file captures the practical checks that have already saved time.

## Current Build Outputs

Run the main generator:

```bash
UV_CACHE_DIR=.uv-cache UV_PYTHON_INSTALL_DIR=.uv-python XDG_CACHE_HOME=.cache \
  uv run python src/enclosure.py
```

The important outputs are:

- `build/sand_cube.step`: enclosure, woofer rear mount, passive radiator mount,
  connector features, fill port, and top reinforcement features.
- `build/jmlc_horn.step`: standalone JMLC/Le Cleac'h horn for the B&C DE250.
- `build/sand_fill_plug.step`: standalone threaded sand plug.
- `build/diagnostics.json` and `build/horn_diagnostics.json`: basic build
  metrics and validity checks.

If Onshape only shows the horn, make sure you imported `sand_cube.step` as well.
The combined horn/cabinet/bracket assembly is not yet a single export.

## Onshape Symptom: Transparent Or One-Sided Horn Surface

Symptoms we saw:

- The horn looked correct from one side but transparent or invisible from the
  other.
- Selecting the inner acoustic face highlighted a face that appeared to belong
  to the outer wall.
- Onshape reported imported-part faults even though local OpenCascade said the
  body was valid.
- A ring or flat shelf appeared visible through the horn wall near the rolled
  mouth.

Most likely causes:

- Flipped or ambiguous face normals in the imported STEP.
- An open surface imported instead of a closed solid.
- STEP entities that Onshape handles poorly for display/import, especially
  `OFFSET_SURFACE` generated by thickening a curved surface.

The known-good horn export that worked in Onshape uses this pattern:

1. Build the horn acoustic surface from the exact JMLC profile.
2. Revolve that acoustic curve into a face.
3. Thicken the revolved face into a printable wall.
4. Convert the resulting horn wall to NURBS/B-spline geometry before STEP
   export.

The later explicit inner/outer meridian-wall version looked mathematically
cleaner, but it reopened the transparent wall issue in Onshape even when the
local OpenCascade checks were clean. The final working hybrid therefore returns
to the thickened-surface wall topology that Onshape liked, then cuts the
adapter/flange with the same JMLC acoustic void instead of a straight cylinder,
so the compression-driver transition stays continuous.

The adapter must meet the horn wall exactly. A small positive radial clearance
in the adapter's JMLC void produced two separate solids: the horn wall and a
valid but disconnected adapter. The downstream `_primary_shape(...)` helper then
kept the larger horn wall and silently discarded the adapter from the export.
Setting that radial clearance to `0.0` produced one valid solid and restored the
adapter in Onshape.

Disabling p-curves reduces duplicate parametric edge data in the STEP file while
preserving the same 3D edge geometry.

## Quick STEP Checks

Check whether the STEP still contains offset-surface entities:

```bash
rg -n "OFFSET_SURFACE|MANIFOLD_SOLID_BREP|ADVANCED_FACE" build/jmlc_horn.step
```

For the current horn, `OFFSET_SURFACE` should not appear after NURBS conversion.

Run the local STEP audit used during debugging:

```bash
uv run python - <<'PY'
from pathlib import Path
from scripts.generate_horn_test_variants import _audit_step
import json

print(json.dumps(_audit_step(Path("build/jmlc_horn.step")), indent=2))
PY
```

Healthy signs:

- `valid: true`
- `self_interference_errors: false`
- `boundary_edge_count: 0`
- `non_manifold_edge_count: 0`
- `counts.solids: 1`
- Horn `face_type_histogram` has no `OffsetSurface`; a mix of `Cylinder`,
  `Plane`, and `BSplineSurface` is expected for the hybrid export.

Warnings:

- `OFFSET_SURFACE` in the STEP: likely Onshape display/import risk.
- Boundary edges above zero: probably an open shell, not a solid.
- Non-manifold edges above zero: likely bad face joining or overlapping faces.
- Negative volume: face orientation or solid orientation is suspect.

## Visual Render Checks

Render local previews with `f3d` before importing into Onshape:

```bash
f3d build/jmlc_horn.step \
  --resolution 1400,1000 \
  --camera-orthographic \
  --camera-view-up=0,0,1 \
  --camera-direction=0,-1,0.25 \
  --camera-zoom-factor=0.8 \
  --background-color=#f4f1ea \
  --color=#777777 \
  --ambient-occlusion \
  --anti-aliasing=ssaa \
  --tone-mapping \
  --grid=0 \
  --axis=0 \
  --filename=0 \
  --notifications=0 \
  --output build/jmlc_horn_side_check.png
```

`f3d` is not a substitute for Onshape import, but it is a fast way to catch
obvious broken geometry before uploading.

## Horn Modeling Rules Learned

- Keep the inner horn acoustic profile exact. It is the thing the sound touches.
- `Solid.thicken(...)` is acceptable for the horn wall only when the result is
  immediately converted to NURBS before STEP export and then verified in
  Onshape.
- The explicit inner/outer meridian-face approach can pass local validity
  checks yet still render as a transparent or one-sided skin in Onshape.
- Keep the adapter acoustic void continuous with the JMLC profile, but do not
  let the adapter correction force a more fragile horn-wall topology.
- Do not add a positive radial clearance between the adapter acoustic void and
  the horn wall. It can prevent the boolean fuse from making one solid.
- Do not blanket-convert the complete horn to NURBS unless a downstream
  importer specifically needs it. Onshape rejected the all-NURBS explicit-wall
  version.
- Use conservative mouth fillets after the main wall is built. Large fillets on
  rolled-back lips are easy to overconstrain.

## Enclosure Modeling Rules Learned

- The woofer is rear-mounted from inside the cabinet through the passive
  radiator service opening.
- The passive radiator is externally mounted on the rear face and needs a flush
  recess plus heat-set insert holes.
- Sand fill ports must feed only the inter-skin void, never the acoustic cavity.
- Connector islands and fill-port tunnels should be solid plastic where they
  pierce the sand void, so sand cannot leak into the speaker volume.
- Keep generated CAD artifacts in `build/`; commit source and docs, not exports.
